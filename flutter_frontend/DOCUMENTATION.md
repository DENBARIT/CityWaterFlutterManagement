Overview

- **Project:** CityWaterFlutter — mobile Flutter app plus an updated Node backend.
- **Purpose:** Client mobile app (Flutter) and backend API for CityWater services.

Prerequisites

- **Flutter SDK:** Install from https://flutter.dev
- **Android SDK / Emulator:** Android Studio with an emulator or a physical device.
- **Xcode (macOS only):** For iOS builds.
- **Node.js & npm:** Install Node 16+ (or LTS) for backend.
- **Prisma CLI (optional):** `npm install -g prisma` or use `npx prisma`.
- **Java JDK:** For Android Gradle builds (OpenJDK 11+ recommended).

Running the Flutter App (mobile)

- From project root run:

```bash
flutter pub get
flutter run -d emulator-5554 --target lib/main.dart
```

- To launch or list emulators:

```bash
flutter emulators
flutter emulators --launch <emulatorId>
```

- VS Code: open the workspace and choose the `Run` target or use the provided task:
  - Task label: `Run CityWater App Emulator` (see .vscode tasks or workspace tasks list).

- Build APK (Android) on Windows:

```bash
flutter build apk --release
# or for debug:
flutter build apk --debug
```

- Files to inspect for app entry and routing:
  - `lib/main.dart` — application entry
  - `lib/screens/` — UI screens
  - `lib/services/` — API/service client code
  - `lib/widgets/` — shared widgets
  - `assets/` — images, icons, sounds, videos

Running the Backend (updated backend)

- Backend location: `updated backend/` (workspace root contains this folder).

- Typical startup steps (from `updated backend/`):

```bash
cd "updated backend"
npm install
# If there is a dev script (check package.json), use:
# npm run dev
# otherwise run directly:
node src/server.js
```

- If the backend uses Prisma (there is a `prisma/` folder and `prisma.config.ts` present):

```bash
# Generate client
npx prisma generate
# Run migrations (development only)
npx prisma migrate dev --name init
```

- Environment variables: create a `.env` file in `updated backend/` (example variables below in Environment & Configuration).

- Backend key files:
  - `updated backend/src/server.js` — main server entry
  - `updated backend/package.json` — scripts and dependencies
  - `updated backend/prisma/` — Prisma schema and migrations
  - `updated backend/src/` — server source code (routes, controllers, etc.)

Environment & Configuration

- Backend `.env` example (in `updated backend/.env`):

```
PORT=3000
DATABASE_URL="postgresql://user:password@localhost:5432/citywaterdb?schema=public"
JWT_SECRET="your_jwt_secret"
NODE_ENV=development
```

- Flutter environment: API endpoints and keys often live in a config file under `lib/config/` or `lib/services/`. Inspect `lib/config/` for concrete variable names.

- Android local.properties: ensure `sdk.dir` is set for local Android SDK (generated by Android Studio). On Windows, `local.properties` may include `sdk.dir=C:\Users\<you>\AppData\Local\Android\sdk`.

Troubleshooting

- Flutter build fails with Gradle/JDK errors:
  - Verify JDK version and `JAVA_HOME` env var.
  - On Windows ensure `local.properties` has correct `sdk.dir`.

- Emulator not found:
  - Run `flutter emulators`; create or start an emulator from Android Studio AVD Manager.

- Backend cannot connect to DB:
  - Confirm `DATABASE_URL` in `.env` and database server accessibility.
  - Run `npx prisma studio` to inspect DB if Prisma is configured.

- Missing npm scripts:
  - Inspect `updated backend/package.json` for available scripts and adjust commands accordingly.
